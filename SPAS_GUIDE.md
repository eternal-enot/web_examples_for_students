# Посібник зі структури та запуску SPA-фреймворків


## 1. Порівняння реалізацій: Базовий підхід vs Фреймворки

### Базовий підхід (`pr3_simple`/`pr3`)
Ця реалізація використовує класичний підхід:
- **Багатосторінковість (MPA)**: «Головна» і «Контакти» існують як фізично окремі HTML-файли (`index.html` та `contacts.html`). Перехід між сторінками перезавантажує браузер.
- **Маніпуляція DOM**: Дані у DOM вставляються імперативно, використовуючи команди на зразок `document.getElementById('...').innerHTML = ...`.
- **Спільний стан**: Усі глобальні змінні (обрана мова, тема, список фільтрів) зберігаються в єдиному `script.js`, що швидко робить код масивним.
- **Відсутність збірки**: Проєкт не потребує компіляції — його можна просто відкрити у браузері прямим кліком по `index.html`.

### Фреймворки (Angular, React, Vue у папках `pr8_*`)
Усі три фреймворки реалізують підхід **SPA (Single Page Application)**:
- **Єдина сторінка**: Сайт завжди має лише один файл `index.html`. Сторінки не перезавантажуються. Натомість, JavaScript непомітно підміняє компоненти на екрані залежно від URL-адреси (використовується *Router*).
- **Компонентна архітектура**: Інтерфейс розбитий на малі, незалежні шматки, які можна перевикористовувати (наприклад: `<Navbar />`, `<ProjectCard />`, `<Footer />`).
- **Реактивність**: Замість ручного пошуку і оновлення елементів, фреймворки використовують *стан (State)*. Якщо ви змінюєте значення змінної "мова", фреймворк сам знаходить усі текстові блоки і миттєво їх оновлює.
- **Етап збірки**: Для роботи потрібен `Node.js`. Код пишеться на TypeScript (`.ts`/`.tsx`) та спеціальних шаблонах (`.vue`), що надалі компілюються збирачем (наприклад, Vite або Angular CLI) у звичайні стислі HTML/CSS/JS файли для публікації.

## 2. Як почати роботу на "чистому" комп'ютері

Якщо ви або хтось інший хочете запустити ці проєкти з нуля на новому комп'ютері, потрібно підготувати середовище:

1. **Встановіть Node.js**  
   Зайдіть на [офіційний сайт Node.js](https://nodejs.org/) і завантажте версію **LTS** (Long Term Support). Разом із нею автоматично встановиться пакетний менеджер `npm`.
   Після встановлення відкрийте термінал і перевірте успішність:
   ```bash
   node -v
   npm -v
   ```
2. **Встановіть редактор коду** (наприклад, VS Code).
3. **Завантажте репозиторій** (за допомогою `git clone <ваш_лінк>`).

> *Самі бібліотеки (React, Vue, Angular) вже прописані у файлах `package.json` і завантажуються автоматично за допомогою команди `npm install`.*

## 3. Запуск проєктів

Кожна папка є ізольованим проєктом. Щоб запустити будь-який із них для перегляду та редагування (з автоматичним оновленням у браузері), виконайте відповідні команди в терміналі.

**Для React-версії:**
```bash
cd pr8_react
npm install   # Завантажить всі модулі (лише 1 раз)
npm run dev   # Запустить локальний сервер розробки
```

**Для Vue-версії:**
```bash
cd pr8_vue
npm install   # Завантажить всі модулі (лише 1 раз)
npm run dev   # Запустить локальний сервер
```

**Для Angular-версії:**
```bash
cd pr8_angular
npm install   # Завантажить всі модулі (лише 1 раз)
npm start     # Запустить Angular сервер на http://localhost:4200
```
*Зупинити сервер можна комбінацією клавіш `Ctrl` + `C` у терміналі.*

## 4. Типова структура проєкту (Що означають файли)

У всіх трьох рішеннях застосовується подібний патерн. Коротко про головні сутності:

* **`package.json`** — "Паспорт" проєкту. Містить назву застосунку, скрипти для запуску (`npm run dev`) та список усіх залежностей (які версії React/Vue потрібні).
* **`node_modules/`** — Системна папка, куди `npm` завантажує пакети. *Ніколи не додається в Git.*
* **`vite.config.ts`** (або `angular.json`) — Конфігураційний файл збирача. Він розповідає системі, як саме слід "склеювати" файли під час релізу.
* **`src/main.tsx`** (або `main.ts`) — Точка входу. Абсолютно найперший файл, який починає виконуватися. Він знаходить `div id="root"` (або app) у `index.html` і каже фреймворку намалювати там сторінку.
* **`src/App.tsx`** (або `App.vue`) — Головний кореневий компонент проєкту. Як правило, саме тут прописаний скелет (`<Navbar>` -> `<Вмісту Сторінки>` -> `<Footer>`).
* **`src/router/`** — Файли (або секції коду), відповідальні за маршрутизацію. Вказують: "якщо в URL адресі написано `/contacts`, слід показати компонент `Contacts`".
* **`src/components/`** — Готові "цеглинки" (компоненти), з яких будуються сторінки (кнопки нагору, вікна проєктів, картки). Такі елементи можна використовувати будь-яку кількість разів на будь-якій сторінці.
* **`src/pages/`** — Комплексні компоненти, які уособлюють цілі сторінки (наприклад, Головна, Контакти). Як правило, вони склеюють між собою дані та менші компоненти.
* **`src/context/`** (або `composables/`, `services/`) — Мозок глобального стану. В цих файлах зберігається логіка для перетворення мови/теми, а також інформація, яка має бути доступна на *будь-якому* рівні проєкту, незалежно від його глибини.
* **`dist/`** — Результат роботи команди `npm run build`. Це повністю готові і найсуворіше мініфіковані `HTML/CSS/JS` файли, які власне і завантажуються на робочий сервер (Hosting/GitHub Pages).
