# Встановлення та налаштування TypeScript з нуля на Windows, macOS і Linux

Цей документ описує базове встановлення та налаштування TypeScript у новому проєкті на Windows, macOS і Linux. Інструкція розрахована на чистий старт і не залежить від попередніх файлів або конфігурацій.

## Windows

### 1. Передумови

Для роботи з TypeScript потрібні такі інструменти:

- `Node.js` - середовище виконання JavaScript
- `npm` - менеджер пакетів, який встановлюється разом із Node.js
- `PowerShell` або `Command Prompt` - для виконання команд

### 2. Встановлення Node.js

Перейдіть на офіційний сайт Node.js:

```text
https://nodejs.org/
```

Завантажте `LTS`-версію для Windows і запустіть інсталятор.

Під час встановлення:

- залиште увімкненим пункт додавання `Node.js` у `PATH`
- підтвердіть встановлення `npm`

Після завершення відкрийте `PowerShell` або `Command Prompt` і перевірте результат:

```bash
node -v
npm -v
```

Якщо команди повертають номери версій, `Node.js` і `npm` встановлено коректно.

### 3. Створення нового проєкту

Перейдіть до каталогу, у якому буде створено новий проєкт:

```bash
cd %USERPROFILE%\Desktop
```

Для `PowerShell` також можна використовувати:

```powershell
cd $HOME\Desktop
```

Створіть окрему папку проєкту та перейдіть до неї:

```bash
mkdir my-typescript-project
cd my-typescript-project
```

Назву `my-typescript-project` можна замінити на будь-яку іншу.

### 4. Ініціалізація npm-проєкту

Створіть базовий файл `package.json`:

```bash
npm init -y
```

Після виконання команди в корені проєкту буде створено файл `package.json`.

Файл `package.json` є основним службовим файлом Node.js-проєкту. Він містить назву проєкту, версію, залежності, dev-залежності та команди в секції `scripts`. Саме цей файл використовується `npm` для керування пакунками та сценаріями запуску.

### 5. Встановлення TypeScript

Рекомендовано встановлювати TypeScript локально в межах конкретного проєкту:

```bash
npm install --save-dev typescript
```

Після встановлення залежностей також буде створено файл `package-lock.json`. Він фіксує точні версії встановлених пакунків і їхніх залежностей. Це потрібно для відтворюваного встановлення однакового середовища на різних машинах.

Перевірте встановлення:

```bash
npx tsc -v
```

### 6. Створення конфігурації TypeScript

Згенеруйте початковий конфігураційний файл:

```bash
npx tsc --init
```

У корені проєкту з'явиться файл `tsconfig.json`.

Файл `tsconfig.json` керує поведінкою TypeScript-компілятора. У ньому задаються правила компіляції, структура каталогів, параметри перевірки типів і перелік файлів, які потрібно включити до збірки.

### 7. Створення базової структури проєкту

Створіть каталог для вихідного коду:

```bash
mkdir src
type nul > src\index.ts
```

Якщо команда `type nul > src\index.ts` не спрацьовує у вашій оболонці, створіть файл вручну в редакторі коду.

Каталог `src` використовується для вихідного коду проєкту. Файл `src/index.ts` у цьому прикладі є початковою точкою програми, тобто місцем, з якого починається виконання прикладу після компіляції.

### 8. Додавання початкового коду

Відкрийте файл `src/index.ts` і додайте:

```ts
const message: string = "Hello, TypeScript!";
console.log(message);
```

### 9. Налаштування `tsconfig.json`

Для базового сценарію файл `tsconfig.json` можна привести до такого вигляду:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src"]
}
```

#### Призначення основних параметрів

- `target` - визначає, у яку версію JavaScript буде компілюватися TypeScript-код
- `module` - визначає формат модулів у згенерованому JavaScript
- `rootDir` - вказує каталог, у якому розміщено вихідні `.ts` файли
- `outDir` - вказує каталог, у який буде записано результати компіляції
- `strict` - вмикає сувору перевірку типів
- `esModuleInterop` - покращує сумісність імпортів між різними модульними системами
- `skipLibCheck` - пропускає перевірку типів усередині зовнішніх бібліотек
- `include` - визначає, які файли або каталоги потрібно включити до компіляції

### 10. Компіляція TypeScript

Запустіть компілятор:

```bash
npx tsc
```

Після цього буде створено каталог `dist`, а в ньому - файл `index.js`.

Каталог `dist` призначений для результатів компіляції. Файл `dist/index.js` є JavaScript-версією `src/index.ts`, яку вже можна запускати через `node`.

### 11. Запуск скомпільованого коду

Виконайте команду:

```bash
node dist/index.js
```

Очікуваний результат:

```bash
Hello, TypeScript!
```

### 12. Налаштування команд у `package.json`

Для спрощення щоденної роботи доцільно додати скрипти:

```json
{
  "name": "my-typescript-project",
  "version": "1.0.0",
  "main": "dist/index.js",
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js"
  }
}
```

Після цього можна використовувати:

```bash
npm run build
npm run start
```

### 13. Опційний запуск TypeScript без попередньої компіляції

Для запуску `.ts` файлів без окремого кроку компіляції можна встановити `ts-node`:

```bash
npm install --save-dev ts-node
```

Після цього доступний запуск:

```bash
npx ts-node src/index.ts
```

### 14. Опційний режим автоматичної компіляції

Для автоматичної перекомпіляції після змін у файлах використовується:

```bash
npx tsc --watch
```

### 15. Приблизна структура готового мінімального проєкту

```bash
my-typescript-project/
├── node_modules/
├── dist/
│   └── index.js
├── src/
│   └── index.ts
├── package.json
├── package-lock.json
└── tsconfig.json
```

### Варіант 2. Встановлення через `apt-get`

#### 1. Оновлення списку пакетів

У Terminal виконайте:

```bash
sudo apt-get update
```

#### 2. Встановлення Node.js і npm

Встановіть `Node.js` і `npm`:

```bash
sudo apt-get install -y nodejs npm
```

Перевірте встановлені версії:

```bash
node -v
npm -v
```

#### 3. Створення нового проєкту

Перейдіть до каталогу, у якому буде створено новий проєкт:

```bash
cd ~/Desktop
```

Якщо каталогу `Desktop` немає, перейдіть до будь-якого іншого зручного каталогу, наприклад:

```bash
cd ~
```

Створіть окрему папку проєкту та перейдіть до неї:

```bash
mkdir my-typescript-project
cd my-typescript-project
```

Назву `my-typescript-project` можна замінити на будь-яку іншу.

#### 4. Ініціалізація npm-проєкту

Створіть базовий файл `package.json`:

```bash
npm init -y
```

Після виконання команди в корені проєкту буде створено файл `package.json`.

Файл `package.json` є основним службовим файлом Node.js-проєкту. Він містить назву проєкту, версію, залежності, dev-залежності та команди в секції `scripts`. Саме цей файл використовується `npm` для керування пакунками та сценаріями запуску.

#### 5. Встановлення TypeScript

Рекомендовано встановлювати TypeScript локально в межах конкретного проєкту:

```bash
npm install --save-dev typescript
```

Після встановлення залежностей також буде створено файл `package-lock.json`. Він фіксує точні версії встановлених пакунків і їхніх залежностей. Це потрібно для відтворюваного встановлення однакового середовища на різних машинах.

Перевірте встановлення:

```bash
npx tsc -v
```

#### 6. Створення конфігурації TypeScript

Згенеруйте початковий конфігураційний файл:

```bash
npx tsc --init
```

У корені проєкту з'явиться файл `tsconfig.json`.

Файл `tsconfig.json` керує поведінкою TypeScript-компілятора. У ньому задаються правила компіляції, структура каталогів, параметри перевірки типів і перелік файлів, які потрібно включити до збірки.

#### 7. Створення базової структури проєкту

Створіть каталог для вихідного коду:

```bash
mkdir src
touch src/index.ts
```

Каталог `src` використовується для вихідного коду проєкту. Файл `src/index.ts` у цьому прикладі є початковою точкою програми, тобто місцем, з якого починається виконання прикладу після компіляції.

#### 8. Додавання початкового коду

Відкрийте файл `src/index.ts` і додайте:

```ts
const message: string = "Hello, TypeScript!";
console.log(message);
```

#### 9. Налаштування `tsconfig.json`

Для базового сценарію файл `tsconfig.json` можна привести до такого вигляду:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src"]
}
```

#### Призначення основних параметрів

- `target` - визначає, у яку версію JavaScript буде компілюватися TypeScript-код
- `module` - визначає формат модулів у згенерованому JavaScript
- `rootDir` - вказує каталог, у якому розміщено вихідні `.ts` файли
- `outDir` - вказує каталог, у який буде записано результати компіляції
- `strict` - вмикає сувору перевірку типів
- `esModuleInterop` - покращує сумісність імпортів між різними модульними системами
- `skipLibCheck` - пропускає перевірку типів усередині зовнішніх бібліотек
- `include` - визначає, які файли або каталоги потрібно включити до компіляції

#### 10. Компіляція TypeScript

Запустіть компілятор:

```bash
npx tsc
```

Після цього буде створено каталог `dist`, а в ньому - файл `index.js`.

Каталог `dist` призначений для результатів компіляції. Файл `dist/index.js` є JavaScript-версією `src/index.ts`, яку вже можна запускати через `node`.

#### 11. Запуск скомпільованого коду

Виконайте команду:

```bash
node dist/index.js
```

Очікуваний результат:

```bash
Hello, TypeScript!
```

#### 12. Налаштування команд у `package.json`

Для спрощення щоденної роботи доцільно додати скрипти:

```json
{
  "name": "my-typescript-project",
  "version": "1.0.0",
  "main": "dist/index.js",
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js"
  }
}
```

Після цього можна використовувати:

```bash
npm run build
npm run start
```

#### 13. Опційний запуск TypeScript без попередньої компіляції

Для запуску `.ts` файлів без окремого кроку компіляції можна встановити `ts-node`:

```bash
npm install --save-dev ts-node
```

Після цього доступний запуск:

```bash
npx ts-node src/index.ts
```
